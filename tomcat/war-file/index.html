Tomcat - simplest .war deployment.



Step-by-step walkthrough to hello world with practically no breath wasted on explanation.
**Please first read how to get started here.

Step 1: Navigate into the project directory:

                    
$> cd /var/lib/tomcat7/webapps/ROOT/;

Step 2: Jar everything into a war file:

                    
$> jar -cvf ROOT.war *;
      #notice how similar "jar" is to "tar".

Step 3: Move the ROOT folder to your home directory. 

$> mv -v /var/lib/tomcat7/webapps/ROOT ~;

Step 4: replace the ROOT folder with the WAR file. 

                    
$> mv ROOT.war /var/lib/tomcat7/webapps/
$> rm -rfv /var/lib/tomcat7/webapps/ROOT
    # !! always check for typos when using rm -rf. XD

Nice things to know

  • A .war file is essentially a zipped folder,
    It can be unzipped with the unzip utility.
  • To deploy a .war file, put it in the "/var/lib/tomcat7/webapps" folder.
    A .war file is equivalent to a project directory, such as the already existing "ROOT".
    You could jar the ROOT folder into ROOT.war, and leave it in the same location, and it will work the same.
  • Here's the expected file hierarchy: 
    └── ROOT (*This project is named "ROOT" but could be anything else)
        ├── META-INF
        │   └── context.xml
        └── WEB-INF
            ├── classes
            │   └── **.class servlets here**
            ├── lib
            │   └── **.jar libs here for project**
            └── web.xml
*If you want to change the name of the project, make sure you update the server config file.
